Why MCP Security Matters

Most organizations deploying AI agents don't realize how exposed their MCP infrastructure really is. Here are the top 5 risks:

  • No Visibility ‚Äî You don't know how many MCP servers you have or which agents can access them
  • Unlimited Access ‚Äî Agents can invoke any tool. If one gets compromised, attackers can do anything

  • No Audit Trail ‚Äî When a breach happens, you have no way to trace who did what
  • Compliance Drift ‚Äî Security policies slowly get ignored as your infrastructure grows

Architecture: Two Simple Components

MCP-G runs two components in your Kubernetes cluster that work together to discover and secure your MCP infrastructure:

1. Governance Controller (Go)

The controller is the brain of MCP-G. It runs as a Kubernetes deployment and continuously monitors your cluster:

  • Discovers all MCP servers from AgentGateway, Kagent, and Agent Registry every 30 seconds
  • Evaluates security posture across 8 governance categories
  • Scores each server on a 0‚Äì100 scale with A‚ÄìF letter grades
  • Exposes results via REST API on port 8090
  • Analyzes risks with AI (optional: Google Gemini or Ollama) for deeper insights

The controller doesn't require any manual input‚Äîit reads your Kubernetes resources and automatically evaluates them.

2. Real-Time Dashboard (Next.js)

The dashboard is a beautiful web interface that lets you see your MCP security posture at a glance:

  • Overview Tab ‚Äî Cluster score, grade, and trend over time
  • MCP Servers Tab ‚Äî List of all discovered servers with individual scores and security gaps
  • Findings Tab ‚Äî Specific security issues with clear fix recommendations
  • Resource Inventory Tab ‚Äî All Kubernetes resources related to your MCP infrastructure

The dashboard auto-refreshes every 15 seconds, so you always see the latest data.

Install in 3 Steps

Step 1: Add the Helm Chart

helm repo add mcp-governance https://charts.techwithhuz.dev
helm repo update

Step 2: Create Namespace and Deploy

kubectl create namespace mcp-governance

helm install mcp-governance mcp-governance/mcp-governance \
  -n mcp-governance

Step 3: Access the Dashboard

kubectl port-forward -n mcp-governance svc/mcp-governance-dashboard 3000:3000

Then open http://localhost:3000 in your browser. That's it! Your MCP servers will be automatically discovered within 30 seconds.

Custom Security Policies (Optional)

You can customize scoring rules by creating a MCPGovernancePolicy in Kubernetes:

apiVersion: governance.mcp.io/v1alpha1
kind: MCPGovernancePolicy
metadata:
  name: my-policy
spec:
  enabled: true
  weights:
    gatewayRouting: 20
    authentication: 20
    authorization: 15
    tlsEncryption: 15
    cors: 10
    rateLimit: 10
    promptGuard: 10
    toolScope: 10
  thresholds:
    minScoreForPassing: 70
    criticalGatewayRequired: true
    tlsRequired: true

Apply it and the controller will immediately start using your custom rules.

Troubleshooting

Dashboard shows no MCP servers?

Check the controller logs: kubectl logs deployment/mcp-governance-controller -n mcp-governance -f

The controller needs RBAC permissions to discover resources. Verify it can list MCP resources: kubectl auth can-i list mcpservers --as=system:serviceaccount:mcp-governance:controller

Dashboard not updating?

Check if the API is reachable: curl http://localhost:8090/api/governance/overview

Also check dashboard logs: kubectl logs deployment/mcp-governance-dashboard -n mcp-governance -f

Best Practices
